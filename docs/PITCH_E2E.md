# E2E сценарий для питча

## Подготовка

1. **Миграция БД и чистый сценарий** (сид очищает чаты, сообщения и лог действий; каталог перелётов/отелей/поездов и пользователи создаются заново):
   ```bash
   npm run db:push
   npm run db:seed
   ```
   После этого можно пройти полный E2E-сценарий с нуля (новый чат, онбординг, варианты, слайдеры, Выбрать/Поделиться).
2. **Тестовые пользователи** (из сида):
   - **+79001234567** — Алексей, кэшбэк 1200 ₽, ~45k ₽/мес, рестораны 35%
   - **+79007654321** — Мария, кэшбэк 4500 ₽, ~120k ₽/мес, люкс 30%
   - **+79001112233** — Дмитрий, кэшбэк 450 ₽, ~28k ₽/мес, бюджетный профиль

3. **Вход без SMS** (development): при запросе кода в ответе приходит `devCode` — введите его в поле «Код из СМС».

## Сценарий питча

1. **Главная** — показать виджет «AI Travel Copilot», нажать → открывается полноэкранный Copilot.
2. **Без входа** — в сайдбаре видно: «Войдите по номеру телефона — мы учтём кэшбэк и траты».
3. **Вход** — нажать «Войти по номеру телефона» в шапке или в сайдбаре → ввести номер (например +79001234567) → получить код (в dev показан `devCode`) → ввести код → войти.
4. **После входа** — в шапке отображается имя пользователя; по клику — кэшбэк на путешествия, средние траты в месяц, траты на рестораны. В сайдбаре Copilot — блок «Контекст пользователя (Т-Банк)» с теми же данными.
5. **Онбординг** — написать направление и даты, например: «Москва — Сочи, 15 июня». Бот на первом этапе уточняет только откуда, куда и даты; бюджет/транспорт/гости — после выдачи вариантов. Можно написать «покажи варианты», чтобы получить подборку по имеющимся данным.
6. **Один маршрут** — после уточнения выдаются 3 карточки «план поездки» (перелёт + отель): **Самая дешёвая**, **Самая быстрая**, **Самая комфортная**. В чате сообщение, что можно дальше уточнять бюджет, транспорт и комфорт. **Мультимаршрут** (несколько городов) — одна карточка с полным сценарием: схема (город → самолёт → город → отель → …), риск, итоговая цена.
7. **Слайдеры** — в сайдбаре блок «Сортировка по предпочтениям»: Скорость, Цена, Комфорт (0–10). При изменении слайдеров порядок карточек в чате и в сайдбаре пересчитывается по предпочтениям.
8. **Карточки** — на каждой карточке плана поездки кнопки **«Выбрать»** и **«Поделиться»** (дизайн Т-Банка).
9. **Лог действий** — в сайдбаре раскрыть «Лог действий (для питча)»: видны события `message_received`, `params_extracted`, `clarification_sent`, `search_completed` с payload.

## Критерии успеха

- Красивый дизайн в стиле Т-Банка (цвета, типографика, карточки).
- Бот корректно обрабатывает любой ввод и добивается недостающих данных.
- Карточки в правильном формате: план поездки (перелёт + отель), метки «Самая дешёвая»/«Самая быстрая»/«Самая комфортная», кнопки Выбрать и Поделиться.
- Слайдеры Скорость/Цена/Комфорт пересортировывают варианты.
- Видно учёт контекста: кэшбэк, средние траты для каждого пользователя.
- Все действия логируются для показа на питче.
