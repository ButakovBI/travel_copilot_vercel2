# Интеграции с внешними сервисами для данных о поездках

## Как устроен поиск маршрутов

Бот **не строит маршруты сам**. Контекст пользователя преобразуется в структурированные запросы к внешним сервисам:

1. **Duffel** — авиабилеты (api.duffel.com; при наличии `DUFFEL_API_KEY`). Маршрут передаётся в IATA-кодах аэропортов; есть маппинг городов (Москва, Сочи, СПб и др.) в IATA. Результаты объединяются с Aviasales/БД.
2. **Aviasales** — авиабилеты (при наличии `AVIASALES_API_KEY` и `AVIASALES_API_URL`).
3. **РЖД** — ж/д билеты (при наличии `RZD_API_URL`; публичного API нет, возможен партнёрский или неофициальный endpoint).
4. **Ostrovok** — отели (при наличии `OSTROVOK_API_KEY` и `OSTROVOK_API_URL`).

Ответы провайдеров **парсятся** в единый формат. Затем выполняется **факт-чек по БД Т-Путешествий**: сопоставление по маршруту/датам и при необходимости уточнение цен. Итоговые варианты отдаются пользователю с пометкой, что данные проверены по базе Т-Путешествий.

Если ключи провайдеров не заданы, поиск идёт по **локальной БД** (те же запросы, данные из seed/импорта). Сценарий работы один и тот же, меняется только источник данных.

Модули: `lib/providers/duffel.ts`, `lib/providers/duffel-iata.ts`, `lib/providers/aviasales.ts`, `lib/providers/rzd.ts`, `lib/providers/ostrovok.ts`, `lib/providers/index.ts` (агрегация + факт-чек).

## Варианты подключения реальных данных

### Duffel (авиа)
- **API:** https://api.duffel.com (Create offer request: `POST /air/offer_requests?return_offers=true`). Документация: [Getting started with flights](https://duffel.com/docs/guides/getting-started-with-flights), [Create offer request](https://duffel.com/docs/api/v2/offer-requests/create-offer-request).
- Тело запроса (как в документации): `data.slices[]` с `origin`, `destination` (IATA), `departure_date` (YYYY-MM-DD); `data.passengers`: `[{ "type": "adult" }]`; `data.cabin_class`: `"economy"`; `data.max_connections`: 2. Если даты нет в запросе, подставляется дата через 7 дней.
- В `.env`: `DUFFEL_API_KEY`. Опционально курсы в RUB: `DUFFEL_GBP_TO_RUB`, `DUFFEL_USD_TO_RUB`, `DUFFEL_EUR_TO_RUB` (по умолчанию 120, 100, 105).
- Альтернатива REST: можно использовать [@duffel/api](https://www.npmjs.com/package/@duffel/api): `duffel.offerRequests.create({ slices: [{ origin, destination, departure_date }], passengers: [{ type: "adult" }], cabin_class: "economy" })`.
- Города преобразуются в IATA через `lib/providers/duffel-iata.ts` (Москва→SVO, Сочи→AER, СПб→LED и др.). Неизвестный маршрут — запрос к Duffel не отправляется.

**Почему маршруты из Duffel могут не подтягиваться:**
1. В `.env` не задан `DUFFEL_API_KEY` или ключ неверный — проверьте переменные и перезапустите сервер.
2. Город не в маппинге IATA — поддерживаются: Москва, Сочи, СПб, Казань, Екатеринбург, Новосибирск, Владивосток, Нижний/Великий Новгород, Оренбург, Калининград, Минск, Баку, Стамбул, Прага, Варна, Подгорица, Пекин, Бангкок и варианты на латинице. Для новых городов добавьте запись в `lib/providers/duffel-iata.ts`.
3. Даты в прошлом или формат дат не принимается API — используйте даты вылета в будущем (YYYY-MM-DD).
4. По маршруту у Duffel нет офферов — тогда в мультимаршруте подставляется оценка цены, а в сайдбаре участок подсвечивается как «без данных».

### Aviasales B2B
- **Документация:** https://b2b.aviasales.ru (API для бизнеса). Требуется регистрация и ключ.
- В `.env`: `AVIASALES_API_KEY`, `AVIASALES_API_URL` (например `https://b2b.aviasales.ru`).

### РЖД
- Официального публичного API для поиска нет. Возможны партнёрские или неофициальные endpoint’ы; при появлении — задать `RZD_API_URL`.

### Ostrovok B2B
- **Подключение:** https://b2b.ostrovok.ru, api@ostrovok.ru. SDK и новая версия API доступны партнёрам.
- В `.env`: `OSTROVOK_API_KEY`, `OSTROVOK_API_URL`.

### Amadeus for Developers

- **Сайт:** https://developers.amadeus.com
- **Возможности:** поиск рейсов, отелей (150 000+ объектов), аэропорты, безопасность. Есть бесплатный self-service tier для разработки.
- **Аутентификация:** OAuth 2.0 Client Credentials.
- **Идея интеграции:** отдельный модуль `lib/integrations/amadeus.ts`, который по запросу копайлота подставляет реальные предложения (или обогащает результаты из БД). Требует регистрации и API-ключей.

### Skyscanner (партнёрский доступ)

- **Документация:** https://developers.skyscanner.net (V3). Старый V2 устарел.
- **Отели:** Hotels Live Prices (реальные цены), Hotels Indicative Prices (оценочные).
- **Ограничение:** доступ по партнёрскому соглашению и API key.

### Собственная синтетическая база

Для полноценного демо без внешних ключей:

1. **Расширить seed:** больше городов (Россия, СНГ, Европа, Турция), отелей с разными звёздами и ценами, рейсов и поездов на разные даты.
2. **Скрипты импорта:** при необходимости подгружать CSV/JSON с отелями и рейсами (см. `scripts/import-offers.md` если есть).
3. **Кэш от внешних API:** в будущем можно один раз выгружать данные из Amadeus/Skyscanner в нашу БД по расписанию и использовать их в копайлоте без реального времени.

## Рекомендация

Для продакшена с реальными ценами и наличием — подключать Amadeus (или партнёрский Skyscanner) через отдельный слой `lib/integrations/` и подмешивать результаты в `lib/search.ts`. До тех пор расширенная синтетическая БД и текущий seed обеспечивают все сценарии бота.
