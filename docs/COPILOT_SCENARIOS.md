# Сценарии Copilot и реализация

Соответствие требований и текущей бизнес-логики.

---

## 1. Онбординг / первый ввод (O1–O6)

| Требование | Реализация |
|------------|------------|
| Не более 5–7 уточняющих вопросов | `MAX_CLARIFICATION_STEPS = 5` в `lib/copilot/params.ts`. После 5 ответов бота переходим к поиску даже при неполных данных. |
| Прогресс онбординга | `getOnboardingProgress(params, missing)` возвращает `{ step, totalSteps, canSkip, missingCount }`. В ответе API передаётся `onboardingProgress` — для прогресс-бара на фронте. |
| Пропустить уточнения, «варианты как есть» | `detectSkipIntent(lastUser)` распознаёт фразы типа «покажи варианты», «пропустить», «достаточно». При таком вводе поиск запускается по текущим параметрам без дополнительных вопросов. |
| Черновик при выходе | Модель `CopilotDraft` в Prisma. API `GET/POST /api/copilot/draft` — сохранение/загрузка по `userId` или `sessionId`. Фронт может вызывать POST при уходе с чата и GET при возврате. |
| 3–5 предварительных вариантов справа | После поиска возвращаются до 5 карточек (`pickTopN(..., 5)`), с метками и риском (`withRouteMeta`). |

---

## 2. Направления и точки маршрута

| Требование | Реализация |
|------------|------------|
| 2–7 городов, порядок или «оптимизировать» | `waypoints` в `ExtractedParams`, описание в EXTRACT_SYSTEM: порядок по умолчанию — «самый быстрый маршрут». |
| Даты, транспорт, бюджет | `dateFrom`, `dateTo`, `transport`, `budgetMin`, `budgetMax` в параметрах и запросах к провайдерам. |
| Нет вариантов → сообщение + рекомендации | В `runSearchViaProvidersWithFallback` при пустом результате — расслабленные параметры; в мультимаршруте в `summaryText` добавляется текст про участки без вариантов и про пересадки. |

---

## 3. Сборка сценариев (3–5 вариантов пакетом)

| Требование | Реализация |
|------------|------------|
| 3–5 целостных вариантов | `pickTopN(rankedByPriority, indices, 5)` + `withRouteMeta(topN)`. |
| Карточки: summary, «Подробнее», «Заменить сегмент», «Сохранить», «Поделиться» | Метаданные в карточке: `routeLabel`, `riskScore`, `riskFactors`. Кнопки — на фронте по этим полям; замена сегмента — отдельный сценарий (чат или API). |
| Индекс риска и пояснение | `computeRiskScore(card)`, `getRiskFactors(card)` в `lib/copilot/scenarios.ts`. В карточки добавляются `riskScore` (0–10) и `riskFactors` (массив строк). |

---

## 4. Сравнение вариантов

| Требование | Реализация |
|------------|------------|
| Ярлыки: Самый дешёвый, самый быстрый, самый комфортный | `withRouteMeta`: первым трём карточкам присваиваются метки `routeLabel`: «Самый дешёвый», «Самый быстрый», «Самый комфортный». |
| Сортировка по приоритету | `sortByPriority(offers, weights)` до ранжирования LLM. Веса задаются через `priorityWeights` в параметрах (0–10: speed, price, comfort). |

---

## 5. Приоритеты (быстрее / дешевле / комфортнее)

| Требование | Реализация |
|------------|------------|
| Слайдеры 0–10: Скорость, Цена, Комфорт | `ExtractedParams.priorityWeights`: `{ speed, price, comfort }`. EXTRACT_SYSTEM просит LLM извлекать приоритет из фраз («дешевле», «быстрее», «комфортнее»). |
| Пересортировка по весам | В `run-v2` перед ранжированием вызывается `sortByPriority(rankedByRelevance, weights)`. Веса по умолчанию: `DEFAULT_PRIORITY` (5, 5, 5). |

---

## 6. Риски и надёжность

| Требование | Реализация |
|------------|------------|
| Индекс риска по варианту | `computeRiskScore(card)` — плейсхолдер (прямой рейс = низкий риск, с пересадкой = выше; мульти-ноги увеличивают риск). |
| Факторы риска, отображение | `getRiskFactors(card)` возвращает короткие строки («Пересадка», «Несколько участков» и т.д.). В карточки кладутся `riskScore` и `riskFactors`. Фильтр по макс. риску и подсветка в детализации — на фронте. |

---

## 7. Пересборка при изменении условий

| Требование | Реализация |
|------------|------------|
| Изменение дат/точек/приоритетов без полного ввода | Пользователь пишет в чат — из диалога снова извлекаются `ExtractedParams` (extract + parse), поиск и ранжирование запускаются заново. |
| «Заменить сегмент» / «замени ночной поезд» | Пока не реализовано отдельным API. Вариант: фронт передаёт в чат запрос «замени участок X на Y» или «замени ночной поезд на дневной автобус», LLM и повторный поиск по участку — в рамках следующей итерации. |
| Undo, превью альтернативы | Не реализовано. Требует хранения предыдущего набора вариантов и API отката. |

---

## 8. Объяснимость и уверенность

| Требование | Реализация |
|------------|------------|
| Резюме выполнимости, компромиссы, красные флаги | LLM возвращает `summary` в ответе ранжирования; в финальном ответе пользователю подставляется этот текст + факт-чек по базе Т-Путешествий. Углублённый блок «почему выполнимо» можно добавить в промпт ранжирования. |

---

## 9. Документы и подготовка к поездке

| Требование | Реализация |
|------------|------------|
| Напоминания о визе/документах/страховке | В БД есть `VisaInfo`; при наличии направления можно подмешивать напоминание в ответ (отдельная доработка). |
| Экспорт маршрута (PDF/ссылка) | Не реализовано. Требует endpoint и генерации PDF или публичной ссылки на маршрут. |

---

## 10. Совместное планирование

| Требование | Реализация |
|------------|------------|
| Поделиться ссылкой (read-only) | Не реализовано. Нужен публичный роут вида `/route/:id` и сохранение сценария по id. |
| Флаг «согласую с кем-то», формат для обсуждения | Не реализовано. Можно добавить в контекст и в ответ бота шаблон текста для согласования. |

---

## Файлы

- **Типы и сценарии:** `lib/copilot/scenarios.ts`, `lib/types.ts` (RouteLabel, OfferCardWithMeta).
- **Параметры и онбординг:** `lib/copilot/params.ts` (MAX_CLARIFICATION_STEPS, getOnboardingProgress, detectSkipIntent, priorityWeights).
- **Поток Copilot:** `lib/copilot/run-v2.ts` (skip, 5 вариантов, приоритет, мета, onboardingProgress).
- **Поиск и ранжирование:** `lib/copilot/search-and-rank.ts` (pickTopN, до 5 индексов), `lib/copilot/tfidf-rank.ts`.
- **Черновик:** `prisma/schema.prisma` (CopilotDraft), `app/api/copilot/draft/route.ts`.
- **Ответ чата:** `app/api/copilot/chat/route.ts` (tripContext, onboardingProgress).
